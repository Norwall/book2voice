# Локальная озвучка книг

Streamlit-инструмент для генерации русских аудиокниг через Silero TTS на CPU.

## Установка

```bat
install.bat
```

Первый запуск скачает Python-зависимости и модель Silero. После этого генерация работает локально из кэша.

## Запуск интерфейса

```bat
run_streamlit.bat
```

Откройте адрес, который покажет Streamlit, загрузите `EPUB`, `TXT` или `FB2`, выберите голос и нажмите `Сгенерировать`.

В блоке `Параметры` можно нажать `Прослушать пример голоса`. Приложение сгенерирует короткий локальный MP3-пример через выбранный голос, частоту, скорость, паузы, размер TTS-фрагмента и число CPU-потоков, а затем покажет встроенный аудиоплеер. Пример содержит целые и порядковые числа, дату, время, дробь, валюту, процент, диапазон, коды и римский номер, чтобы на слух проверить нормализацию. Он не сохраняется в папку результата книги.

Книга делится на выходные главы по расчетной длительности: по умолчанию около 20 минут на файл. Граница ставится по окончанию предложения, а при расчете учитываются выбранная скорость и паузы между TTS-фрагментами. Главы сохраняются как `001.mp3`, `002.mp3`, `003.mp3` и так далее. Если включить `Склеить в одну книгу`, рядом появится `book.mp3`.

По умолчанию к папке результата добавляется дата и время. Это защищает старые результаты от перезаписи. Если дату и время отключить и выбрать папку, где уже есть `001.mp3`, `002.mp3` или `book.mp3`, генерация не начнется и приложение попросит выбрать пустую папку.

Во время генерации доступна кнопка `Стоп`. Остановка выполняется безопасно: текущий фрагмент озвучки или операция конвертации завершается, после чего генерация прекращается. В параметрах можно выбрать, что делать после остановки: оставить уже готовые главы или удалить папку результата. Автоматическое удаление доступно только для новой папки, созданной текущим запуском; уже существующие пользовательские папки приложение не удаляет.

После первого готового TTS-фрагмента интерфейс показывает примерное оставшееся время и ожидаемое время окончания. Оценка рассчитывается по скорости текущего запуска, уточняется после каждого фрагмента и учитывает фактическую скорость MP3-конвертации после завершения первой главы.

Кнопка `Открыть папку с книгой` доступна, как только создан каталог результата. Ею можно пользоваться во время озвучки и после завершения, чтобы сразу открыть готовые главы в Проводнике Windows. Если при остановке выбрано удаление результата, кнопка снова станет недоступна после удаления папки.

Перед передачей текста в Silero приложение нормализует символы, которые часто встречаются в FB2/EPUB/TXT и могут ломать TTS: неразрывные пробелы, типографские тире и кавычки, служебные Unicode-символы, emoji, приватные Unicode-символы и ударения над буквами. Например, скрытая метка ударения `U+0301` удаляется, чтобы модель не падала с пустым `KeyError`.

### Озвучивание чисел

Silero `v5_5_ru` удаляет цифры из входного текста, поэтому приложение по умолчанию преобразует числовые записи в русские слова до расчета длительности, разбиения на фрагменты и синтеза. Для преобразования используется локальная зависимость `num2words`; сетевые запросы во время генерации не нужны.

Поддерживаемые примеры:

| Исходный текст | Текст для Silero |
| --- | --- |
| `Глава 1. Часть II.` | `Глава первая. Часть вторая.` |
| `5 марта 2025 года` | `пятое марта две тысячи двадцать пятого года` |
| `3:46` | `три часа сорок шесть минут` |
| `21,5` | `двадцать одна целая пять десятых` |
| `25 ₽, скидка 12%` | `двадцать пять рублей, скидка двенадцать процентов` |
| `5–7` | `от пяти до семи` |
| `код 007, телефон 911` | `код ноль ноль семь, телефон девять один один` |
| `1-й вариант` | `первый вариант` |

Переключатель `Озвучивать числа` действует и на книгу, и на кнопку `Прослушать пример голоса`. Если его выключить, числовые записи передаются без преобразования и Silero может их пропустить. Нормализация использует детерминированные правила, а не полный морфологический анализ русского предложения, поэтому неоднозначные номера и редкие форматы лучше заранее записывать словами.

Семантические ссылки на сноски FB2/EPUB вида `[1]` удаляются отдельно, чтобы номера ссылок не прерывали повествование. Обычный текст в квадратных скобках без признака ссылки сохраняется.

Если Silero сообщает, что фрагмент слишком длинный, приложение автоматически делит только проблемный TTS-фрагмент на меньшие части и повторяет синтез. Это помогает при случаях, когда текст укладывается в лимит символов, но внутри модели разворачивается в последовательность длиннее технического лимита. Дополнительные паузы между такими аварийными подфрагментами не добавляются.

Если генерация завершится с ошибкой, интерфейс покажет текст причины и блок `Подробности ошибки` с технической диагностикой. Невидимые символы в проблемном фрагменте выводятся в виде `\uXXXX`, чтобы их можно было найти в исходном тексте.

Основные параметры интерфейса:

- `Длительность главы, минут` задает расчетную длину каждого MP3-файла, по умолчанию 20 минут.
- `Размер фрагмента TTS, символы` ограничивает технические куски текста для Silero и не управляет длиной выходных MP3-глав. Если отдельный кусок все равно превышает внутренний лимит модели, он будет автоматически разделен и озвучен повторно.
- `Озвучивать числа` включает преобразование числовых записей в слова. Отключайте его только для возврата к прежнему поведению, при котором Silero пропускает цифры.
- `Прослушать пример голоса` создает тестовый MP3 по текущим параметрам, чтобы проверить выбранный голос до генерации книги.
- `Скорость` и `Пауза между фрагментами` учитываются при расчете длительности глав.

## Запуск из командной строки

```bat
python generate_book.py book.epub --voice baya --out outputs\my_book
python generate_book.py book.txt --voice xenia --chapter-minutes 20 --merge
python generate_book.py book.fb2 --voice baya --out outputs\fb2_book
python generate_book.py book.fb2 --no-normalize-numbers
```

Параметр `--chapter-minutes` задает расчетную длительность выходной MP3-главы. Если параметр не указан, используется 20 минут.

Флаги `--normalize-numbers` и `--no-normalize-numbers` управляют озвучиванием чисел из командной строки; по умолчанию преобразование включено.

Доступные голоса: `aidar`, `baya`, `kseniya`, `xenia`, `eugene`.

Если запуск из командной строки завершится ожидаемой ошибкой, например из-за отсутствующего файла, занятой папки результата или поврежденной книги, программа выведет короткое сообщение `Error: ...` и завершится с кодом `1`.

## Проверка

```bat
python -m unittest discover -s tests -q
python -m compileall audiobook_tts tests -q
python -m py_compile streamlit_app.py generate_book.py
```

Тесты проверяют подготовку текста для Silero, преобразование чисел, удаление ссылок на сноски, регрессию на FB2-книге с неразрывными пробелами и скрытыми Unicode-ударениями, декодирование TXT, автоматическое дробление слишком длинных TTS-фрагментов, защиту папки результата и восстановление поврежденного кеша модели.

## Примечания

- Для MP3 используется FFmpeg из пакета `imageio-ffmpeg`; отдельная установка FFmpeg не нужна.
- TXT поддерживает распространенные кодировки `UTF-8`, `UTF-16`, `CP1251` и `KOI8-R`.
- TXT и FB2 без явной структуры тоже делятся на выходные главы по расчетной длительности.
- Кнопка `Стоп` работает в интерфейсе Streamlit. Запуск из командной строки завершается обычным прерыванием процесса.
- Если файл модели Silero в `.models\silero` поврежден, приложение переместит его в файл с суффиксом `.corrupt` и один раз попробует скачать модель заново.
- Для длинных книг CPU-генерация может занимать часы.
